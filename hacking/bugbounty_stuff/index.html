You may only redirect to *.bugbountyhunter.com Open URL Redirect

The application at https://www.bugbountytraining.com/challenges/challenge-7.php hosts a redirect link on the page. That redirect link is controlled by the parameter ?redirectUri=

When checking for filters with the parameter it is clear that / \ and ? are filtered completely out, but that # causes the application to modify the link in an unexpected way. If the test was ?redirectUri=https://webhook.site#.bugbountyhunter.com the link turns into https://www.bugbountyhunter.com, rather than simply deleting the #1c86ee

It can then be deduced some application logic failure is happening within the first domain we are attempting to redirect to, and after using %E3%80%82 we can encode a . to achieve the redirect.
